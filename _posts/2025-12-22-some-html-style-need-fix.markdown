HTML 本身是基于西方文字开发的超文本语言，所以主要用于从内容上区别文字的用途。为了形象显示某些文字特点，所以会为其内容呈现一定的在展示格式。例如为了强调文字，使用加粗方法——<b>强调</b>文字。这在超文本语言中，使用了`<b>强调</b>`代码。如果使用 Markdown 代码，就是`**强调**`，显示如**强调**。这没有问题。

但是，在英文中还有一种突出和强调作用是使用意大利体——我们平时也称其为斜体，实际应用中，经常使用标签`em`，如超文本标识方法——`<em>强调</em>`，显示为<i>强调</i>，使用 Mrakdown，就是 `*强调*`。这种显示在汉文中是没有的！如果汉字的强调使用了斜体，就很不伦不类，甚至看其来很费力，根本起不到强调的作用。而在汉文中有一种强调是在字的下面加点，所以可以通过使用下面的 CSS 来将英文中的 `em` 标签实现这种显示，呈现为*强调*。而需要使用斜体的使用具有相同显示效果的 `i` 标签即可。如果是非常重要可以使用三个`***`同时实现加粗和加点——***非常重要***！

~~~css
em{
font-style: normal;
text-emphasis: dot;
text-emphasis-position: under;
}
~~~

还有，在 Markdown 的段落文字内部，回行默认为连接在一起的文字，这在英文中，是使用空格连接词语的，所以回行的文字默认用空格连接。但是在汉文中用空格连接回行的文字就会多出了一个空格。所以在汉文书写中就不能再使用回车回行了。

例如：“千山万水”这个词，如果在中间加一个回车回行，就变成“千山
万水”，可以看到，中间有了一定的空隙。这就是 Markdown 转换器会在回行的词语中间自动用空格加以连接。所以，如果使用汉语编写文字，就不要进行回车回行。

还有在中西文混用情况下，中西文中间如果不加空格会显得很拥挤，如果加一个空格有些时候会显得很散，这个人为加入的空格有些过长，这个就没有办法了，聊胜于无吧。很多文字编辑软件都会在中西文混排时自动加入 1/8 空格优化显示。现在，新的浏览器标准也加入了此功能。所以可以直接输入不用考虑空格的问题了。但是目前还不是所有的浏览器都实现了这个功能。

现代汉文文献中，段落的首行通常应该缩进两个字符的位置。这也需要进行特别设定：

~~~css
p {
    text-indent: 2em;
}
~~~

所以，如果使用汉文写作，需要人为的注意一些显示问题。
